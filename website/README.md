# Orkestra Docs (Docusaurus v2)

A Crossplane-style documentation site with **Orkestra purple branding**
(`#6B3FA0`). Content lives in the repo-level `docs/` folder; this `website/`
folder is just the Docusaurus app.

## Why Docusaurus v2 (over VitePress)

- First-class **versioning** (`docs-version` plugin) with a built-in version
  switcher — matches Crossplane's API reference UX directly.
- Built-in **right-hand sticky TOC**, collapsible sidebar with auto-generation
  from folder structure, Algolia DocSearch integration, and an i18n pipeline.
- Larger plugin ecosystem (local search, sitemap, OpenAPI generators) so the
  "API reference first" feel is achievable with config, not custom React.

## Quick start

```bash
cd website
npm install
npm run start      # dev server on http://localhost:3000
npm run build      # static site -> website/build
npm run serve      # preview the production build
```


## Replacing the logo

Drop your SVG/PNG at `website/static/img/logo.svg` (same filename). For a
different filename, update `themeConfig.navbar.logo.src` in
`docusaurus.config.js`.

## Customizing the purple theme

All color tokens live in `website/src/css/custom.css`. Adjust
`--ifm-color-primary` and the `--orkestra-purple-*` scale to taste. Dark-mode
overrides are in the `[data-theme='dark']` block.

## Search

- **Algolia DocSearch**: replace `YOUR_APP_ID` / `YOUR_SEARCH_API_KEY` /
  `indexName` in `docusaurus.config.js` (`themeConfig.algolia`).
- **No Algolia keys?** The `@easyops-cn/docusaurus-search-local` plugin is
  already wired in and indexes everything at build time — works offline.
  Remove the `algolia` block if you only want local search.

## Versioning

```bash
# Snapshot current docs as v1.0:
npm run docusaurus docs:version 1.0
```

This creates `website/versioned_docs/`, `website/versioned_sidebars/`, and
adds `"1.0"` to `versions.json`. The navbar version switcher appears
automatically.

## Validating / regenerating

```bash
npm run validate-docs   # fails if any md is missing a title
npm run gen-sidebar     # writes sidebars.generated.js from docs/ tree
```

## Deploy — GitHub Pages

1. Set `url` / `baseUrl` / `organizationName` / `projectName` in
   `docusaurus.config.js`.
2. Add the workflow at `.github/workflows/deploy.yml` (provided).
3. In repo settings → Pages, set source = "GitHub Actions".
4. Push to `main`. CI builds and deploys.

Manual one-shot: `GIT_USER=<you> npm run deploy`.

## Deploy — Cloudflare Pages

Project settings:
- **Build command:** `cd website && npm install && npm run build`
- **Build output directory:** `website/build`
- **Root directory:** repo root
- **Node version:** 18 or 20

## Accessibility checklist

- ✅ Semantic landmarks (`<nav>`, `<main>`, `<aside>`) provided by theme
- ✅ Sidebar fully keyboard-navigable (Tab / Enter / Space / Esc to collapse)
- ✅ Skip-to-content link rendered by Docusaurus
- ✅ Color-contrast: purple `#6B3FA0` on white passes WCAG AA for normal text
- ✅ Dark-mode primary `#a17ec9` chosen for AA contrast on `#14101c`
- ✅ Search input has visible focus ring + aria-label
- ✅ All code blocks include language label + copy button with aria-label

## SEO

- `sitemap.xml` auto-generated by `@docusaurus/preset-classic`
- `robots.txt` shipped in `static/`
- Per-page `<title>` and `description` from frontmatter
- Open Graph + Twitter Card meta via `themeConfig.image`
- Canonical URLs derived from `url` + `baseUrl`
